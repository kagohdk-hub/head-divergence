目的
    一つのvscode window ないで public info と private info を編集しつつ
    repo は分けたい
アイディア
    public-repo のフォルダ内に
    private-repo のフォルダを作成し
    public-repo フォルダ直下の .gitignore に private-repo/ を記載することで
    public-repo からは無視されるようにする
    そうして private-repo のフォルダ内にまた .git を作れば
    同じ repo ないに public と private を共存できる。
不採用理由
    repo のトップのレイヤーがズレる
    これは認知負荷を高める結果になると判断した。
