# 現段階評価
-   出力としては、とても良い。  
    記法は、ただ列挙しただけになっているため、判断基準などをあらかじめ共通化しておくなどの工夫がまだ必要。
-   deepcopyも関数の中だけで使えば、おそらく、その関数のスコープを抜けた時にメモリ解放してくれるはず。

# 考察
-   ソースコードは、時系列がなく、広げるモード的思考様式  
    output.mdは、時系列で表示され、まとめるモード的思考様式  
    1つのことを2つの様式で、それらが連携した形で表示できる。  
    （ただし、広げるモード→まとめるモード　の順でしか書けないが）
    -   まずソースコードも時系列で書いて、一旦動かす。  
        その後、ソースコードを分岐に整理していく。  
        さらにその後、ソースコードの分岐ごとに検討をし、新たな分岐を増やしたり、不要な分岐を減らしたりする。  
        都度、動かして全体像を見る。

# 展望、活用

-   if分岐のイメージは、  
    分岐ロジック自体をsenario_01nのような命名の関数で記載し、それを外側から引数でフルパス通すようなイメージ。  
    ```python
    def main():
        def senario_01n(condition: bool):
            if condition is True:
                print("pattern A")
            else:
                print("pattern B")
        for condition in [True, False]:
            senario_01n(condition)
    ```
